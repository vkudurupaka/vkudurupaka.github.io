4. What is the latest Kubernetes version?
Kubernetes releases three minor versions per year. The latest version is typically around v1.30+ (verify at kubernetes.io).

EKS usually supports up to Kubernetes 1.29 / 1.30 depending on region. Always verify using AWS documentation.
6. What is QoS in Kubernetes?
QoS (Quality of Service) defines how Kubernetes prioritizes Pods during resource pressure. There are three QoS classes:
  • Guaranteed
  • Burstable
  • BestEffort
7. Which QoS class is evicted last?
Guaranteed Pods are evicted last during resource starvation.
8. How do you configure requests and limits to make a Pod evicted last?
Set CPU and memory requests equal to limits: resources: requests: cpu: "500m" memory: "256Mi" limits: cpu: "500m" memory: "256Mi"
9. What is a liveness probe?
Checks if a container is running. If it fails, Kubernetes restarts the container. livenessProbe: httpGet: path: /health port: 8080
10. What is a readiness probe?
Checks if a container is ready to receive traffic. If it fails, the Pod is removed from service endpoints.

15. How would you see all CloudWatch logs in one place?
Use centralized logging:
  • Cross-account subscription filters
  • Kinesis Firehose
  • Central logging account
  • OpenSearch or S3 + Athena
16. Can a Lambda in Account A send logs to Account B?
Yes. Use CloudWatch subscription filters with cross-account permissions or Kinesis Firehose.
17. How would you centralize logs from multiple Lambdas?
All Lambdas log to CloudWatch → Subscription filters → Central account → Store in S3/OpenSearch.

19. If a developer manually changes a Lambda tag and you run terraform plan, what happens?
Terraform detects drift and shows changes in the plan. On apply, it will revert the manual change. To avoid this:
  • Restrict console access
  • Use IAM/SCP policies
  • Enable drift detection in CI/CD
  • Use AWS Config rules
